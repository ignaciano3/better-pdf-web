# better-pdf 1.4.0 features in the web editor

Date: 2026-06-28
Status: Approved (design)

## Context

`@ignaciano3/better-pdf` was bumped `1.3.0 → 1.4.0` (the modification already
present in `package.json` / `bun.lock`). The 1.4.0 changelog adds:

- **More field metadata on read** — `FieldInfo` now exposes `multiline`, `comb`,
  `password`, `editable`, `align`, `tooltip`, `defaultValue`, and
  `fontName`/`fontSize`.
- **Widget visibility flags on read** — each widget carries `hidden`, `print`,
  `noView` (from the annotation `/F` flags).
- **Writable default value (`/DV`)** — builder options `defaultValue` (text),
  `defaultChecked` (checkbox), `defaultSelected` (radio/dropdown/listbox); plus
  setters on existing fields. This is the value a viewer's "Reset form"
  restores, independent of the current value (`/V`).
- **Writable password text fields** — `addTextField({ password: true })` sets the
  `/Ff` Password flag so viewers mask the displayed value.
- **Form reset** — `form.resetField(name)` / `form.reset()` restore fields to
  their `/DV` (or clear them).
- **Fix: generated form fields now print** — created widgets set the `/F` Print
  flag.

This app is an authoring + export tool: it reads AcroForm fields from an uploaded
PDF, lets the user edit/place them, and **rebuilds** a fresh document on export
(re-authoring every field via `createForm()`). It does not have a form-fill
runtime.

### How each changelog item maps to this app

| 1.4.0 item | Action |
| --- | --- |
| Password text fields | Implement — authoring + UI + import. |
| Writable `/DV` defaults | Implement — authoring + UI + import. |
| Richer field metadata on read | Implement — import mapper. |
| Widget visibility flags on read | Implement — used to **skip** hidden/noView fields on import (see below). |
| Fix: generated fields print | **No code** — automatic from the version bump. |
| `form.reset()` / `resetField()` | **No UI surface** — fill-time viewer operation; this app authors/exports. Covered transitively: the `/DV` defaults we add are exactly what reset (and a viewer's "Reset form" button) act on. |

## Approach

Extend the three existing seams in place and surface new controls in the existing
modal — no new components or modes:

1. `FieldElement` model — `src/lib/pdf/types.ts`
2. Authoring mapper — `src/lib/pdf/build.ts` (`authorField`)
3. Import mapper — `src/routes/editor/extract-fields.ts`
4. UI — `src/routes/editor/FieldPropertiesModal.svelte`

Rejected alternatives: a separate "Advanced" modal/tab (the modal isn't crowded
enough to warrant the split); a new fill/reset mode to host `form.reset()` (out
of scope — the app does not fill forms).

## Section 1 — Data model (`types.ts`)

Add to `FieldElement`, each with JSDoc matching neighboring fields:

- `password?: boolean` — text fields only. Maps to the lib's `password` option
  (`/Ff` Password flag); display masking only.
- `defaultValue?: string` — text fields: the `/DV` reset default, distinct from
  `value` (`/V`, the initial value).
- `defaultChecked?: boolean` — checkbox `/DV`.
- `defaultSelected?: string` — radio / dropdown / combo / listbox `/DV`; must be
  one of the field's `options` (or radio states).

`value` retains its current meaning (initial `/V`).

## Section 2 — Authoring (`build.ts` `authorField`)

- text → pass `password: true` when set; pass `defaultValue` when non-empty
  **and** within `maxLength` (guard mirroring the existing `comb` guard so an
  over-long default cannot fail the export).
- checkbox → `defaultChecked: true` when set.
- radio / dropdown / combo / listbox → `defaultSelected` only when it is present
  in the resolved `options` (guard against the lib's validation throw).

No change needed for the print fix — it is automatic from the version bump.

## Section 3 — Import / richer read (`extract-fields.ts`)

Extend `FieldInfoLike` with the new read fields: `multiline`, `comb`, `password`,
`editable`, `align`, `tooltip`, `defaultValue` (string | null), `fontSize`
(number | null), and per-widget `hidden` / `print` / `noView`.

In `fieldInfoToElement`:

- Carry over `tooltip`, `align`, `fontSize`, and (text only) `multiline`,
  `comb`, `password`.
- Map an `editable` dropdown to our `combo` kind (a plain dropdown stays
  `dropdown`).
- Map `defaultValue` to `defaultValue` (text), `defaultSelected` (choices, only
  when it is one of `options`/states and not `Off`), or `defaultChecked`
  (checkbox: true when the default is an on-state, i.e. not null/`Off`).
- **Skip a field whose first widget is `hidden` or `noView`.** The export rebuild
  re-authors every field as visible + printing, so importing a hidden field would
  silently turn it visible. Skipping is the faithful behavior. Documented as a
  known limitation.
- `fontName` is intentionally dropped — the app does not author field value fonts
  by `/DA` resource name.

`comb` is carried only when `maxLength` is present (the model/authoring invariant
already requires it).

## Section 4 — UI (`FieldPropertiesModal.svelte`)

For every value-bearing type, expose an **Initial** (`/V`) and a **Reset
default** (`/DV`) control:

- text:
  - Rename the existing "Default value" input to **Initial value** (binds
    `value`).
  - Add **Reset default** input (binds `defaultValue`).
  - Add a **Password (mask input)** checkbox alongside Multiline / Comb.
- checkbox:
  - **Checked initially** (binds `value` — non-empty ⇒ checked).
  - **Checked by default** (binds `defaultChecked`).
- radio / dropdown / combo / listbox:
  - **Initial selection** select (binds `value`).
  - **Reset default** select (binds `defaultSelected`).
  - Both sourced from `options` with a "— none —" entry that clears the binding.

Empty/none selections delete the corresponding optional property rather than
storing an empty string, matching the existing modal idioms.

## Section 5 — Tests

Extend existing suites (follow current patterns):

- `extract-fields.test.ts` — new metadata round-trips, editable→combo mapping,
  default mapping per type, hidden/noView skip.
- `build.field.test.ts` — `password` flag, the three default kinds, and the
  guards (over-long text default, default-selected not in options).
- `FieldPropertiesModal.svelte.test.ts` — new controls render and bind for each
  field type.

Only touched files are formatted with prettier (the repo is not prettier-clean
repo-wide).

## Known limitations

- Hidden / no-view fields are dropped on import (cannot be re-authored hidden by
  the builder).
- `fontName` (`/DA` resource font) is not preserved on import.
- `form.reset()` / `resetField()` have no in-app surface (no fill runtime); the
  `/DV` defaults authored here are what a viewer's reset acts on.
