# better-pdf 1.4.0 features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface better-pdf 1.4.0's password text fields, writable `/DV` reset defaults, and richer field-metadata-on-read in the web editor (model, authoring, import, UI).

**Architecture:** Extend the three existing seams in place — the `FieldElement` model (`types.ts`), the authoring mapper (`build.ts` `authorField`), and the import mapper (`extract-fields.ts`) — and add controls to the existing `FieldPropertiesModal`. No new components or modes.

**Tech Stack:** SvelteKit (Svelte 5 runes), TypeScript, `@ignaciano3/better-pdf@1.4.0`, Vitest + @testing-library/svelte. Use **bun** for all commands (never npm/pnpm).

## Global Constraints

- Package manager is **bun** — run tests with `bun run test:unit -- --run <path>`.
- Library is `@ignaciano3/better-pdf@1.4.0` (already in `package.json`).
- Exact 1.4.0 builder option names: text `password?: boolean`, `defaultValue?: string`; checkbox `defaultChecked?: boolean`; radio/dropdown/listbox `defaultSelected?: string`; dropdown `editable?: boolean`.
- Exact 1.4.0 `FieldInfo` read fields: `defaultValue: string | null`, `password`, `multiline`, `comb`, `editable`, `align: 'left'|'center'|'right'`, `tooltip: string | null`, `fontSize: number | null`; per-widget `hidden`, `print`, `noView` (booleans).
- `value` on `FieldElement` means the initial `/V`; the new `defaultValue`/`defaultChecked`/`defaultSelected` mean the `/DV` reset default — keep them distinct.
- "Reset form" (`form.reset()`) is a fill-time viewer op with no in-app surface; do NOT add a reset UI. The `/DV` defaults are what it acts on.
- The "generated fields now print" fix needs no code — it is automatic from the version bump.
- Format only touched files with prettier (`bunx prettier --write <files>`); the repo is not prettier-clean repo-wide.
- Coordinates: editor uses top-left origin; the existing helpers handle the flip — do not change coordinate logic.

- [ ] **Step 4: Implement the mapper changes**

In `fieldInfoToElement`, change the `kind` binding from `const` to `let` and add the hidden-skip + editable→combo right after the widget guard:

```ts
	let kind = toFieldKind(info.type);
	if (!kind) return null;
	const widget = info.widgets[0];
	if (!widget) return null;
	// Hidden / no-view widgets can't be re-authored hidden by the builder; the
	// rebuild would turn them visible, so skip them on import.
	if (widget.hidden || widget.noView) return null;
	// An editable dropdown is our `combo` kind.
	if (kind === 'dropdown' && info.editable) kind = 'combo';
```

Then, just before `return element;`, add the metadata carry-over:

```ts
	if (info.tooltip) element.tooltip = info.tooltip;
	const isValueText =
		kind === 'text' || kind === 'dropdown' || kind === 'combo' || kind === 'listbox';
	if (isValueText) {
		if (info.align && info.align !== 'left') element.align = info.align;
		if (info.fontSize != null && info.fontSize > 0) element.fontSize = info.fontSize;
	}
	if (kind === 'text') {
		if (info.multiline) element.multiline = true;
		if (info.password) element.password = true;
		if (info.comb && info.maxLength != null) element.comb = true;
	}
	// Reset default (/DV) → per-kind default property. "Off"/empty are sentinels.
	const dv = info.defaultValue;
	if (dv != null && dv !== '' && dv !== 'Off') {
		if (kind === 'text') element.defaultValue = dv;
		else if (kind === 'checkbox') element.defaultChecked = true;
		else if (kind === 'radio') {
			if (info.states.includes(dv)) element.defaultSelected = dv;
		} else if (kind === 'dropdown' || kind === 'combo' || kind === 'listbox') {
			if (info.options.includes(dv)) element.defaultSelected = dv;
		}
	}
```
